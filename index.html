Planilla caso de uso 1: Hacer consulta

Caso de uso Hacer consulta
Identificador CU001
Actores Usuario registrado / no registrado
Tipo Primario
Referencias ---
Precondiciones Ninguna
Postcondiciones Ni idea
Descripción El usuario entra a la primera interfaz para poder realizar la consulta
Resumen ----
Flujo normal
Nro Actor Sistema
1 Entra al sitio web
2 Muestra el chat
3 Aparece un pop up, inhabilitando todas las otras funciones a excepción de los botones “Iniciar sesión” y “Registrarse”, que muestra las categorías a seleccionar:
- Cocina
- Plomería
- Electricidad
Cada uno con un color e icono que los represente
4 Elige una de las categorías
5 Cierra el pop up
6 Habilita el chat
7 Hace la pregunta
8 Confirma si la pregunta está relacionada con la categoría
9 Si la pregunta no está relacionada con la categoría, entonces se le muestra un mensaje “No podemos responder esa pregunta, haz otra ☹”. Si la pregunta está relacionada entonces pasa al paso 10
10 Envía la solicitud a la API de OpenAI para que el modelo responda la pregunta
11 La API devuelve una respuesta
12 Procesa los resultados
13 Muestra en el chat la respuesta
14 Vuelve al punto 4
Flujo alternativo
10 Envía la solicitud a la API de OpenAI para que el modelo responda la pregunta
10.1 Fallo en el recibimiento de la petición a la API de OpenAI
10.1.1 Se muestra mensaje “Intente más tarde”
